04 / 12
The master AI layer

api/chat.js, up close

One endpoint. One Claude tool use loop. It can cross six or more data sources in a single question and gates every write behind your approval.

Streaming by default

It streams text, thinking, and live tool steps over SSE. First token in about half a second.

Model by effort

Haiku for fast turns, Sonnet by default, Opus for heavy reasoning with adaptive thinking.

Approval gated writes

Writes return a code, batch into one "do X and Y, go?", and fire on your yes. Reads run immediately.

The two real gaps

It has no memory across turns, and only one of its action executors actually fires on approval today. Those are the things to fix.

The one decision

Open by default, dormant until you call it

You do not need a cold start. The cockpit stays open on your main PC, and a low power dormant layer simply listens for the wake. A clap plus "Ryujin Activate" brings the full experience forward. No full system start, no separate boot.

OPTION A
Cold start each time

Boot the app on every wake. Slow, and the operating system suspends a backgrounded or closed tab. The trap to avoid.

OPTION B
Separate OS daemon

A background process that survives reboot and screen off. Heavier, and only needed if the browser throttles the mic.

RECOMMENDED
Dormant layer in the open cockpit

The cockpit stays open and foreground on your main PC. A low power listener idles, then activates on the wake. Lightest path.

The call

Cockpit open by default on your main PC, with a dormant sleep layer, a low power clap plus wake word listener, that brings it forward on command. A tiny local helper at Code/jarvis-daemon is the only fallback if the browser ever suspends the microphone in the background.

Upgrade the brain

Faster to feel, smarter to trust

Biggest speed lever: default to Haiku

Run conversational turns on the fast model and escalate only when a task needs it. This is pure config.

Biggest feel lever: semantic memory

After each turn, store a few fact bullets and a decision log, then feed them back. This is what makes it remember you.

Reach every page: a Cmd-K bar

One command bar over the existing endpoint, on every cockpit page, so the assistant is one keystroke away.

No rewrite needed

All five upgrade phases are config changes or thin additions over a tested endpoint. The chat loop itself stays as is.

11 / 12
Honest read

What is real, what is hype

Build now

Speak the briefing, the Cmd-K bar, model tiering, and hands free read only queries. Cheap, and high impact.

Finicky, set expectations

Wake reliability in a real house, voice accuracy on jargon and addresses, and robotic narration without a polish pass.

Discount the hype

It is 55 tools, not 82. Wake to speech is one to two seconds, not instant. And the dormant wake layer is still real software to keep running.

Do not build a second action system, a full Electron app, or a cold start boot on every wake. The chat loop plus its tools is already the action layer.

The sequence

Cheap wins first, wake layer last

  1. Model tiering and cache audit. Speed and cost win, in config. Effort S
  2. Speak my briefing button. Proves the whole voice loop end to end. Effort S
  3. Push to talk command bar, Cmd-K. Operate from any page. Effort M
  4. Wire the missing approval executors. Close the silent no op gap. Effort S to M
  5. Semantic memory. The part that makes it remember you. Effort M
  6. Dormant wake layer. Clap plus "Ryujin Activate" from the always open cockpit. Effort M
