How it works

One file, one engine, hard rules baked in

Each deck is one HTML page

A stack of section.slide blocks plus presentation.js at the bottom. The engine auto-activates: arrows or swipe to present, S for scroll, F for fullscreen.

Build with /ultraslide

Ask in the working session. It copies this kit, writes public/deck-<slug>.html, adds a card to decks.html, then runs the deploy ritual.

Hard rules

Navy and gold for internal decks, no em dashes, real numbers only, every claim a real asset, node check the inline script, codex before merge, manual prod deploy, curl smoke.

10 / 11
The review loop

Suggest in place, then save-refresh

Press N on any slide to leave a note

Notes stick per slide, and when you open the deck logged into Ryujin they sync to the server so Cat can leave them too. Yellow notes are yours, gold are coaching, blue are revised copy.

Then run /ultraslide refresh <slug>

It reads every note off the deck, applies them as edits, re-verifies the render, redeploys, and clears the notes it addressed. Your suggestions become the new deck.

Recurring asks become memory

If the same edit keeps coming up (a phrasing, a layout rule), the refresh writes it to durable memory so the next deck is built that way from the start.

Reference · pick the right palette

Three audiences, three looks

INTERNAL
Navy + gold

Ops reviews, SOPs, strategy, design reviews. This kit. #0e1a35 + #facc15.

CREW / ADMIN
Navy + teal-mint

Crew-facing portals and tools. #0e1a35 + #2dd4bf / #0ea5e9.

CUSTOMER
Cream + royal blue

Anything a homeowner sees. Jewels visual rules, lock-in CTA, never em dashes.

The instant tell

Mixing an internal palette into a customer deck, or the reverse, reads as a template seam. Pick one for the whole deck and hold it.
