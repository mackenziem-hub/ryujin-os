04 / 09
Move 1 · the number one leak

Make internal shipping automatic so built work stops dying in the queue

The fleet can build far faster than you can merge. Right now you are the only outlet, and the gate that was supposed to drain internal PRs got deleted in a rewrite.

  • Root: the Jun-16 router rewrite stripped the autonomous internal-deploy ship-gate from foreman.md. Result: 17 PRs stuck, you as the sole valve.
  • Fix A: re-instate the internal ship-gate as a deterministic Stop hook that runs the prod smoke-check. Anthropic's exact pattern. A future rewrite can never strip a hook.
  • Fix B: back-pressure. Foreman stops dispatching new builds past about 4 un-deployed READY branches. Drain before you fill.
  • Fix C: one deploy-queue card in the Admin pillar showing internal READY-but-undeployed PRs and the client-facing ones waiting on you.
  • Keep the wall: internal ships itself, client-facing (proposals, portals, funnels) stays your button.

Decide

Re-arm the autonomous internal ship-gate as a Stop hook, internal-only, client-facing deploys still yours. Yes drains the 17-PR backlog and keeps it drained.

05 / 09
Move 2

Claim the work, not the order, so two machines stop colliding

Anthropic names tightly-coupled work on shared code as the single worst fit for multi-agent. That is exactly the floating-picker setup that duped four PRs in a day.

  • Root: two machines on one queue converge on the same item by identical "highest-value thing" reasoning. The claim-guard protects the order id, not the actual work.
  • Fix A: split lanes by disjoint subsystem and files (proposals, leads, infra, sales-ops), not "whoever's free grabs the next order."
  • Fix B: atomic claim-on-pull, a compare-and-set on the bus, so two machines physically cannot grab the same work.
  • Fix C: every order carries boundaries, the files NOT to touch. Anthropic's named anti-dup fix, and the cheapest one to add.

Decide

Lanes split by subsystem with a hard work-claim, or keep the floating picker. The lane split is the lower-token, lower-conflict path and ends the recurring dup class.

Moves 3 and 4 · the infrastructure

Collapse to one bus and turn the hardest rules into hooks

Two of the leaks share a root: orders and rules both live in OneDrive prose that forks, clobbers, and gets silently rewritten. Make them durable.

  • One bus: order payloads still live in OneDrive inbox JSON (desk F has 6 forked inboxes, zombie orders 88 to 91 hours old) while state lives on /api/fleet. Move orders onto the Blob bus, retire the OneDrive critical path, add order TTL so zombies auto-cancel, one claims ledger.
  • Rules as hooks: your firmest invariants (no edits to sent proposals, no settings.json writes, no-send-without-signoff, the ship-gate) live as prose the model drifts from. Encode them as PreToolUse and Stop hooks. Deterministic, cannot regress in a rewrite.
  • Sync and prune: command files are not synced across machines, so foreman already diverged HAL vs Mind-Palace. MEMORY.md is 57KB and only about 46% loads. Sync the command files, prune CLAUDE.md and MEMORY.md to evergreen rules plus pointers, demote procedure into skills.

Decide

You apply the desk.md and foreman.md bus-rewire edits. The harness blocks me from self-editing the running skill, so this one needs your hand or a permission rule.

Moves 5 and 6

Verify the metric before acting, and summon the fleet only when it's worth 15x

Two cheaper disciplines. One stops the fleet acting on lies, the other stops it spending like an audit on a typo.

  • Ground truth: the snapshot keeps drifting from reality and health reads 10/10 anyway. A false-zero lead count drove a whole "lead-starved" strategy across Operator, Oracle, and Foreman. Make the health check assert section presence and freshness, never let a writer co-mutate a section, and verify a metric on prod before acting on it.
  • Right-size: Anthropic measured multi-agent at about 15x the tokens of a single chat, worth it only for high-value breadth. Run the full Operator to Foreman to Desks fan-out for audits, multi-PR waves, and sweeps. A one-line fix is a single Claude, not the loop. Scheduled data refreshes are workflows in code, not agents.
  • Credits: the credit-wall is a standing single point of failure with no alarm. Top it up and add a balance check that pings before zero.

Decide

Top up Anthropic credits to revive the hosted layer (Oracle, the Z-fighters, the do-it-for-me drive). It is the only thing on this deck that cannot be live-verified at $0.

08 / 09
Prompting · orders as code

The order format that makes every task self-finishing

Most of the above is structure. This is the prompt-level change that makes each desk finish clean without you babysitting it.

  • Four-part schema on every order: objective, output format, allowed tools and sources, and boundaries (what's out of scope, which files NOT to touch). Anthropic's exact spec. The boundaries line is also the anti-dup fix from Move 2.
  • End every order with a verification step: the explicit "prove it works on prod" line. Anthropic calls this the single highest-leverage upgrade to an order format.
  • Show evidence, not assertions: make curl output, the exercised endpoint, or a screenshot a required field of done. "Looks done" is not done.
  • Machine-readable checklist: a passes true/false JSON the desks update but cannot delete, for multi-session build waves.
  • Scope the reviewer: tell the fresh-context reviewer to flag only correctness and requirements gaps, not style. Stops the over-engineering your own rules already warn against.
