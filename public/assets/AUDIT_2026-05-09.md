## Fallback pattern reference

Three patterns used across the codebase:

### 1. Image with sibling text fallback (img → span)
```html
<img src="/assets/logo/wordmark.png" alt="RYUJIN OS"
     onerror="this.style.display='none';this.nextElementSibling.style.display='inline'">
<span class="nav-name" style="display:none">RYUJIN OS</span>
```

### 2. Image preferred, inline SVG default-visible (img onload swap)
Used for nav icons. SVG visible by default (no flash), PNG hides SVG once loaded.
```html
<img src="/assets/images/icon-tickets.png" alt="" style="display:none"
     onload="this.style.display='block';this.nextElementSibling.style.display='none'">
<svg viewBox="0 0 24 24"><!-- always-visible default --></svg>
```

### 3. CSS layered backgrounds
Browser silently skips missing layers — gradient or solid color always shows through.
```css
background:
  url('/assets/textures/app-bg.png') center/cover no-repeat,   /* canonical */
  url('/assets/textures/storm-bg.jpg') center/cover no-repeat, /* existing */
  radial-gradient(...);                                          /* final fallback */
```

### 4. Video with multiple sources
Browser tries each `<source>` in order until one plays.
```html
<video autoplay muted loop playsinline poster="/assets/textures/storm-bg.jpg" onerror="this.style.display='none'">
  <source src="/assets/videos/hero-bg-loop.mp4" type="video/mp4">    <!-- canonical -->
  <source src="/assets/videos/ocean-caustics-bg.mp4" type="video/mp4"> <!-- existing -->
</video>
```
