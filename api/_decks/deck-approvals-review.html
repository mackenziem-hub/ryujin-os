08 / 09
Why this happened

The approve button never ran the action

  • Only send-email executes. The other 12 write actions have no executor, so approving them returns nothing and the row flips back to pending.
  • Pending means it re-queues. Each agent run that wants the same action adds another copy, which is how one ticket became five and one payment email became twelve.
  • The portal approve button does nothing. Even send-email is lost if approved from the page rather than from chat, because that path never reads the action payload.
  • The fix: wire the executors (create ticket, move pipeline, add note, GHL task) to the actions that already exist, collapse the two execution paths into one, and dedupe before queueing. This is Tier 2 item 1 from the infra audit.
