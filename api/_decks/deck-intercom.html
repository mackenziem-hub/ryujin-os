The idea in one line

One event in. The right work out, to the right people.

The tools already existed: tasks, work orders, paysheets, the inbox, the routing brain. What was missing was the connective tissue: the choreography that ties an event to a fan-out. That is what got built.

The principle

A job signs, a quote comes in, materials run short: the system places everyone in advance. You are left with the two things only you can do, verify the work and record the videos.

05 / 10
The front of the funnel

A quote request becomes a proposal in motion.

The moment a real quote request lands (Instant Estimator or Revive estimator), a "Prepare video proposal" task fires to Cat, and you see it. The two of you build the presentation. The lead already pinged you, so nothing waits on a manual hand-off.

Guarded on purpose

Only the two quote-request sources trigger it, so a soft PDF lead does not bury Cat in proposal tasks. Diego stays out, he handles booked inspections, a separate event.

Under the hood

How it is configured.

Small, reusable pieces. Each event endpoint makes one call; the heavy lifting lives in shared libraries that reuse the app's own APIs, so numbering and validation are handled.

07 / 10
Why you can trust it

Built so it can never hurt a signing.

Fail-soft

The acceptance is saved first. If any part of the fan-out hiccups, it is logged and skipped, the customer's signing always succeeds.

Idempotent

A re-accept never double-fires, it matches the exact job-specific task title before creating anything.

Controlled-tested

A live test caught a silent bug that would have made every real signing fan out nothing. Found and fixed before it could bite.
