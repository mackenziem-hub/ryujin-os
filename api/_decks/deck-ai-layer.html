Layer 1 · Agentic toolchain

How Ryujin gets built

Boris loop

Verification-target-first coding: a pass or fail target before any code, plan review, then an adversarial codex review gate before deploy.

Plan mode

Multi-step plans are written and saved as artifacts before implementation. 22 plans on file, refined before a line of code commits.

UltraSlide

This engine. Internal decks built from one HTML file with a sticky-note review loop synced to Supabase, plus a headless render verifier.

HyperFrames

HTML to deterministic, frame-perfect MP4 with GSAP animation, authored by the agent. New this week, proven on the Revive price-drop ad.

Ponytail

A six-skill simplification suite that audits for over-engineering and deletes it, reaching for the standard library before custom code.

Connected tools

Eight connected services through MCP: Gmail, Drive, Calendar, Higgsfield, Canva, Stripe, Zapier, plus DeepWiki repo Q&A.

Layer 1 · The second brain

Context that survives every session

The Obsidian _brain vault

The single source of truth for working state. Session context, decisions, and what is pending, all in one navigable vault.

Durable auto-memory

Timeless principles captured as 345 linked topic files, indexed and auto-loaded at the start of every session so lessons never get re-learned.

Unified LOAD and SAVE

One command pulls live state from six sources at session start: tickets, approvals, hot deals, decks, appointments, and finance. One command saves the handoff.

Agent-native vault

New this week: Obsidian CLI and Bases skills let the agent read, write, and query the vault directly instead of by hand.

Layer 2 · Customer intelligence

Knowing every customer

CUSTOMER 360
One state bundle

Lifetime value, repeat count, review-eligibility alerts, and a years-since-last-job churn histogram in a single payload.

RECONCILIATION
Daily revenue audit

A cron agent cross-checks estimates, work orders, paysheets, and GHL won deals, flagging drift by 11 identity rules.

LTV + CHURN
Re-roof projection

Projects lifetime value and re-roof timing by customer-age cohort to surface who is due to come back.

Job identity engine

Records merge only on surname AND street number, never a single name, so two different Smiths never collapse into one customer.

Layer 2 · Sales and quoting

From lead to signed, priced live

Quote Engine v3.1

Tiered roof, exterior, metal, and flat quotes from merchant-backed line items, built guided in five questions or fully overridden.

Multi-offer compare

Gold, Platinum, and Diamond side by side from a single set of measurements, so customers compare tiers live on the proposal.

Instant estimator

A self-serve roof estimate that captures a GHL lead and opportunity, routes it to a pipeline, and triggers follow-up.

Custom Scope Proposal Generator

Hand-built, single-use proposals with a custom scope grid, exclusions, and redeck notices, accepted with an e-signature.

Proposal PDF renderer

Share-gated client proposals rendered to print-perfect PDF via headless Chromium, with the envelope frozen at send.

Layer 2 · Marketing intelligence

Every ad dollar, tracked toward a job

Meta live refresh

Campaigns, spend, impressions, clicks, and leads pulled from the Meta Graph API every morning before the agents run.

KPIs and ROAS

Joins ad spend with Ryujin estimates and attribution for blended and ad-sourced ROAS, CAC per campaign, and full funnel progression.

Intelligence dashboard

Live funnel, cost-per-lead and ROAS, attention alerts for token expiry or zero-lead spend, and top ads by performance.

Social planner pipeline

Clip to caption to schedule across Facebook, Instagram, YouTube, TikTok, and Google Business Profile, with status reconciled from GHL.

Campaign control

Pause and resume campaigns, change daily budgets, and manage ad-set status and retargeting audiences from inside Ryujin.

Layer 2 · Autonomous ops

The system runs while you sleep

Roughly 76 autonomous passes a day, today, before any opt-in agents are switched on.

Daily agent suite

Four agents each morning analyze the pipeline, ops, comms, and ad ROI, storing findings for the owner briefing.

Morning and evening briefing

A structured ritual: KPI scouter, pipeline pulse, ops status, comms, and the top three actions for the day.

Daily snapshot cache

The shared intelligence store, fed by every agent, that powers the dashboards, the briefing, and the state endpoints.

Inbox triage

Reads GHL conversations every 20 minutes, classifies each message, and fires a high-signal SMS only for active leaks or buying intent.

In progress

The Production Pipeline agent (job-stage suggestions) and the Quest Scanner (auto-filled daily to-dos) are built and wired, pending a per-tenant opt-in flag.
